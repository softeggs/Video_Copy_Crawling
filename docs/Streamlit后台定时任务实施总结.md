# Streamlit 后台定时任务实施总结

## 实施目标

在 Streamlit Web 应用中集成后台定时任务功能，让用户可以直接在浏览器中启动/停止定时任务，无需单独运行脚本。

## 实施内容

### 1. 核心模块

#### `core/background_scheduler.py` - 后台任务管理器

**主要功能**:
- 单例模式管理后台任务
- 线程安全的启动/停止控制
- 状态查询和结果记录
- 手动触发执行

**关键特性**:
- ✅ 使用守护线程（daemon thread）
- ✅ 独立的事件循环（asyncio）
- ✅ 异常自动恢复
- ✅ 线程安全（threading.Lock）

**主要方法**:
```python
class BackgroundScheduler:
    def start(check_interval: int) -> Dict
    def stop() -> Dict
    def get_status() -> Dict
    def execute_once() -> Dict
```

### 关键技术点

#### 1. 单例模式

```python
class BackgroundScheduler:
    _instance = None
    _lock = threading.Lock()
    
    def __new__(cls):
        if cls._instance is None:
            with cls._lock:
                if cls._instance is None:
                    cls._instance = super().__new__(cls)
        return cls._instance
```

**优点**:
- 确保只有一个实例
- 避免重复启动
- 线程安全

#### 2. 守护线程

```python
self.thread = threading.Thread(
    target=self._run_scheduler,
    daemon=True  # 守护线程
)
```

**优点**:
- 主程序退出时自动结束
- 不阻塞主线程
- 资源自动清理

#### 3. 独立事件循环

```python
def _run_scheduler(self):
    loop = asyncio.new_event_loop()
    asyncio.set_event_loop(loop)
    try:
        # 使用新的事件循环
        result = loop.run_until_complete(...)
    finally:
        loop.close()
```

**优点**:
- 不干扰主线程的事件循环
- 支持异步操作
- 独立管理

#### 4. 异常恢复

```python
while self.is_running:
    try:
        # 执行任务
        ...
    except Exception as e:
        logger.error(f"任务异常: {e}")
        time.sleep(60)  # 等待后继续
```

**优点**:
- 单次失败不影响后续执行
- 自动恢复
- 记录错误日志

## 注意事项

### 1. 资源占用

- 后台线程占用少量内存（~10-50MB）
- 处理任务时占用 CPU 和网络
- 建议根据服务器性能设置间隔

### 2. 并发问题

- 同一个 Streamlit 实例只能运行一个后台任务
- 多个 Streamlit 实例会有多个后台任务
- 不建议同时运行多个任务（可能重复处理）

### 3. 稳定性

- Streamlit 重启会停止后台任务
- 需要手动重新启动
- 生产环境建议使用独立脚本

### 4. 日志管理

- 后台任务的日志写入 `logs/app.log`
- 建议定期清理日志文件
- 可以在 Web 界面查看最近日志

## 已知限制

1. **依赖 Streamlit**
   - 必须保持 Streamlit 应用运行
   - 重启应用需要重新启动任务

2. **单实例限制**
   - 一个 Streamlit 实例只能运行一个后台任务
   - 不支持多任务并行

3. **状态持久化**
   - 状态保存在内存中
   - 重启应用会丢失状态

4. **浏览器依赖**
   - 需要通过浏览器操作
   - 不支持命令行控制
