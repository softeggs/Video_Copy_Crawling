# 原始转录 AI 润色功能说明

## 功能概述

定时任务新增了"原始转录 AI 润色"子任务，自动检测并重新处理标签中含有"原始转录"的记录，为它们补充 AI 润色。

## 问题背景

### 为什么会有"原始转录"记录？

在视频处理过程中，可能因为以下原因导致 AI 润色失败：

1. **网络问题**
   - 网络不稳定
   - 连接超时
   - 请求失败

2. **API 问题**
   - API 服务繁忙
   - API 额度不足
   - API 响应超时

3. **系统问题**
   - AI 处理超时（超过 180 秒）
   - 系统资源不足
   - 其他异常

当 AI 润色失败时，系统会：
- ✅ 保留原始转录文本
- ✅ 标记标签为"原始转录"
- ✅ 状态设置为"已完成"
- ✅ 不影响其他处理步骤

## 功能说明

### 自动检测

定时任务会自动扫描飞书表格，查找满足以下条件的记录：

1. ✅ 标签字段包含"原始转录"
2. ✅ 处理状态为"已完成"
3. ✅ 有详细内容（原始转录文本）

### 处理流程

```
1. 检测到"原始转录"记录
   ↓
2. 更新状态为"AI润色中"
   ↓
3. 读取原始转录文本
   ↓
4. 调用 AI 进行润色
   ↓
5. 生成新的内容：
   - 一句话总结
   - 核心观点
   - 纠错后的详细内容
   - 金句
   - 新标签（AI 生成）
   ↓
6. 过滤标签：
   - 确保移除"原始转录"标签
   - 使用 AI 生成的新标签
   ↓
7. 更新飞书记录
   ↓
8. 生成新的 Markdown 文件
   ↓
9. 状态更新为"已完成"
```

### 处理特点

**只做 AI 润色，不重新下载和转录**

- ✅ 节省时间（不需要重新下载视频）
- ✅ 节省资源（不需要重新转录音频）
- ✅ 保留原始数据（标题、作者、链接等）
- ✅ 只更新 AI 生成的内容

**更新的字段**：
- 一句话总结
- 核心观点
- 详细内容（纠错后）
- 金句
- 标签（移除"原始转录"）
- 笔记路径
- 处理时间

**保留的字段**：
- 标题
- 作者
- 原始链接
- 发布日期

## 常见问题

### Q1: 为什么我的记录没有被处理？

**检查清单**：
1. 标签中是否包含"原始转录"
2. 处理状态是否为"已完成"
3. 详细内容字段是否有内容
4. 定时任务是否正在运行

### Q2: 处理失败后会怎样？

A: 
- 状态恢复为"已完成"
- 标签保持"原始转录"
- 记录错误信息
- 下次检查时会再次尝试

### Q3: 会重新下载视频吗？

A: 不会。只进行 AI 润色，不重新下载和转录。

### Q4: 原始转录文本会丢失吗？

A: 不会。AI 润色后的文本会替换详细内容字段，但原始文本已保存在 Markdown 文件中。

### Q5: 可以手动标记记录为"原始转录"吗？

A: 可以。在飞书表格中手动添加"原始转录"标签，系统会在下次检查时处理。

### Q6: 处理需要多长时间？

A: 取决于文本长度和 AI 响应速度，通常 30-180 秒。

### Q7: 会创建重复记录吗？

A: 不会。系统会更新现有记录，不会创建新记录。

### Q8: 可以批量处理吗？

A: 可以。定时任务会自动处理所有"原始转录"记录。
