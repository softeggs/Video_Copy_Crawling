# AI 超时配置说明

## 概述

AI 处理超时配置允许你根据实际情况调整 AI 处理的最大等待时间。

## 默认配置

- **超时时间**: 180 秒（3 分钟）
- **重试次数**: 3 次
- **总计最长等待**: 180s × 3 = 540 秒（9 分钟）

## 配置方法

### 方法 1: 环境变量配置（推荐）

在 `.env` 文件中添加：

```env
# AI 处理超时时间（秒）
AI_TIMEOUT=180
```

**常用配置：**

```env
# 快速模式（适合短视频）
AI_TIMEOUT=60

# 标准模式（推荐）
AI_TIMEOUT=180

# 长视频模式
AI_TIMEOUT=300

# 超长视频模式
AI_TIMEOUT=600
```

### 方法 2: 代码配置

如果不设置环境变量，系统会使用默认值 180 秒。

## 超时机制说明

### 工作流程

1. **第 1 次尝试**: 等待最多 180 秒
   - 成功 → 返回结果
   - 超时 → 进入第 2 次尝试

2. **第 2 次尝试**: 等待最多 180 秒
   - 成功 → 返回结果
   - 超时 → 进入第 3 次尝试

3. **第 3 次尝试**: 等待最多 180 秒
   - 成功 → 返回结果
   - 超时 → 跳过 AI 处理，使用原始文本

### 失败后的处理

如果 3 次尝试都超时：
- ✅ 不会中断整个流程
- ✅ 自动跳过 AI 润色
- ✅ 使用原始转录文本
- ✅ 继续生成 Markdown 和同步飞书

## 如何选择超时时间

### 考虑因素

1. **视频长度**
   - 短视频（< 1 分钟）: 60-120 秒
   - 中等视频（1-5 分钟）: 120-180 秒
   - 长视频（5-15 分钟）: 180-300 秒
   - 超长视频（> 15 分钟）: 300-600 秒

2. **AI 提供商**
   - OpenAI: 通常较快，120-180 秒足够
   - Gemini: 可能较慢，建议 180-300 秒
   - 中转 API: 取决于中转服务质量

3. **网络状况**
   - 网络良好: 使用默认值
   - 网络不稳定: 适当增加 30-60 秒

4. **内容复杂度**
   - 简单内容: 使用默认值
   - 复杂内容（专业术语多）: 增加 60-120 秒

### 推荐配置

| 场景 | 超时时间 | 说明 |
|------|---------|------|
| 快速测试 | 60s | 适合短视频测试 |
| 日常使用 | 180s | 推荐配置，适合大多数场景 |
| 长视频 | 300s | 适合 10 分钟以上的视频 |
| 稳定优先 | 600s | 网络不稳定或内容复杂时 |

## 性能优化建议

### 1. 减少超时的方法

**优化转录文本长度：**
- 使用更小的 Whisper 模型（tiny, base）
- 转录文本会更短，AI 处理更快

**选择更快的 AI 提供商：**
- OpenAI 通常比 Gemini 快
- 使用质量好的中转 API

**优化提示词：**
- 简化提示词可以加快响应

### 2. 避免超时的方法

**分段处理：**
- 将长视频分段处理
- 每段独立进行 AI 分析

**使用更强大的模型：**
- GPT-4 比 GPT-3.5 更稳定
- Gemini 2.0 比 1.5 更快

**检查网络：**
- 使用稳定的网络连接
- 避免高峰期处理

## 监控和调试

日志会显示每次尝试的耗时：

```

准备重试 AI 处理...
AI 处理尝试 2/3
AI 处理成功（尝试 2/3）
```

### 常见问题

**Q1: 为什么会超时？**

A: 可能的原因：
- 视频太长，转录文本过多
- 网络不稳定
- AI 服务负载高
- 中转 API 响应慢

**Q2: 超时后会丢失数据吗？**

A: 不会。超时后会：
- 自动重试（最多 3 次）
- 最终失败后使用原始文本
- 不影响其他处理步骤

**Q3: 可以设置无限超时吗？**

A: 不推荐。建议设置合理的超时时间（如 600 秒），避免长时间等待。

**Q4: 超时时间越长越好吗？**

A: 不是。超时时间应该根据实际需求设置：
- 太短：容易超时失败
- 太长：等待时间过长，影响体验

**Q5: 如何知道设置多少合适？**

A: 建议方法：
1. 先使用默认值 180 秒
2. 观察日志中的实际处理时间
3. 如果经常超时，增加 60-120 秒
4. 如果从不超时，可以适当减少

## 示例配置

### 场景 1: 快速处理短视频

```env
# .env
AI_TIMEOUT=60
WHISPER_MODEL=tiny
AI_PROVIDER=openai
```

**特点：**
- 快速响应
- 适合 1-3 分钟的短视频
- 牺牲一些精度换取速度

### 场景 2: 标准配置（推荐）

```env
# .env
AI_TIMEOUT=180
WHISPER_MODEL=base
AI_PROVIDER=openai
```

**特点：**
- 平衡速度和质量
- 适合大多数场景
- 推荐日常使用

### 场景 3: 高质量长视频

```env
# .env
AI_TIMEOUT=300
WHISPER_MODEL=medium
AI_PROVIDER=openai
OPENAI_MODEL=gpt-4-turbo-preview
```

**特点：**
- 高质量输出
- 适合 10-20 分钟的视频
- 处理时间较长

### 场景 4: 网络不稳定环境

```env
# .env
AI_TIMEOUT=600
WHISPER_MODEL=base
AI_PROVIDER=openai
```

**特点：**
- 容错性强
- 适合网络不稳定的环境
- 等待时间较长
