# AI 失败自动跳过功能说明

## 功能概述

当 AI 润色功能启用但处理失败时（如 API Key 无效、网络问题、配额用尽等），系统会自动跳过 AI 处理，使用原始转录文本继续完成流程，确保整个处理不会因为 AI 问题而中断。

## 工作原理

### 处理流程

```
视频下载 → 音频提取 → 语音识别 → AI 处理 → 生成笔记 → 飞书同步
                                    ↓ (失败)
                                自动跳过 AI
                                使用原始文本
```

### 失败检测

系统会捕获以下类型的 AI 处理错误：
- API Key 无效或过期
- 网络连接问题
- API 配额用尽
- 服务不可用
- 超时错误
- 其他 AI 服务异常

### 降级策略

当 AI 处理失败时：
1. **记录警告日志**：记录失败原因
2. **显示友好提示**：告知用户 AI 处理失败，使用原始文本
3. **使用基础处理**：调用 `_create_basic_content` 方法
4. **继续后续流程**：生成笔记、飞书同步等正常进行

## 代码实现

### Pipeline 中的实现

```python
# 4. AI 处理（如果启用）
if self.ai_processor.enable_polish:
    self._update_progress(progress_callback, "🤖 AI 分析与润色...")
    try:
        processed_content = await self.ai_processor.process(
            transcription['text'],
            metadata,
            progress_callback
        )
    except Exception as ai_error:
        # AI 处理失败，自动跳过并使用原始文本
        logger.warning(f"AI 处理失败，跳过 AI 润色: {str(ai_error)}")
        self._update_progress(progress_callback, "⚠️ AI 处理失败，使用原始文本...")
        
        # 使用基础处理（不经过 AI）
        processed_content = self.ai_processor._create_basic_content(
            transcription['text'],
            metadata
        )
else:
    self._update_progress(progress_callback, "📝 整理内容...")
    processed_content = await self.ai_processor.process(
        transcription['text'],
        metadata,
        progress_callback
    )
```

### 基础内容处理

`_create_basic_content` 方法会：
1. 清理转录文本
2. 提取前 200 字作为摘要
3. 简单分段作为核心观点
4. 提取关键词作为标签
5. 使用视频标题

## 使用场景

### 场景 1：API Key 未配置

**情况**：
- 用户启用了 AI 润色
- 但未配置或配置了无效的 API Key

**结果**：
- ⚠️ 显示警告：AI 处理失败，使用原始文本
- ✅ 流程继续，使用语音识别的原始文本
- ✅ 生成基础格式的笔记

### 场景 2：网络问题

**情况**：
- API Key 正确
- 但网络连接不稳定或无法访问 AI 服务

**结果**：
- ⚠️ 显示警告：AI 处理失败（网络问题）
- ✅ 自动降级到基础处理
- ✅ 完成整个流程

### 场景 3：API 配额用尽

**情况**：
- API Key 有效
- 但已达到使用配额限制

**结果**：
- ⚠️ 显示警告：AI 处理失败（配额用尽）
- ✅ 使用原始文本继续
- ✅ 不影响其他功能

## 优势

### 1. 提高可靠性
- 不会因为 AI 问题导致整个流程失败
- 确保用户至少能得到语音识别结果

### 2. 更好的用户体验
- 友好的错误提示
- 自动降级，无需用户干预
- 流程不中断

### 3. 灵活性
- 可以在 AI 服务不稳定时继续使用
- 适合测试和开发环境
- 降低对 AI 服务的依赖

### 4. 成本控制
- API 配额用尽时仍可使用基础功能
- 避免因 AI 失败浪费已完成的处理步骤

## 配置

### 启用/禁用 AI 润色

在应用界面：
```
侧边栏 → AI 设置 → ☑️ 启用 AI 润色
```

在代码中：
```python
pipeline = ProcessingPipeline(
    ai_provider='gemini',
    enable_ai_polish=True,  # True: 启用（失败时自动跳过）
                            # False: 禁用（直接使用原始文本）
    whisper_model='base'
)
```

## 最佳实践

### 1. 配置有效的 API Key
虽然有降级机制，但建议配置有效的 API Key 以获得最佳体验。

### 2. 监控日志
定期检查日志中的 WARNING 信息，了解 AI 失败频率。

### 3. 测试环境
在测试环境中可以故意使用无效 API Key 来测试降级功能。

### 4. 用户提示
在界面上明确告知用户 AI 处理状态，避免困惑。

## 未来改进

### 可能的增强

1. **重试机制**
   - AI 失败时自动重试 1-2 次
   - 使用指数退避策略

2. **多 AI 提供商切换**
   - 主 AI 失败时自动切换到备用 AI
   - OpenAI ↔ Gemini 自动切换

3. **部分降级**
   - 某些 AI 功能失败时只降级该功能

4. **用户选择**
   - 让用户选择失败时的行为
   - 重试 / 跳过 / 中止
