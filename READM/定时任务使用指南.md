# 飞书表格定时任务使用指南

## 功能概述

定时任务会自动检查飞书多维表格中的**空白链接**，并自动执行完整的处理流程（下载、转录、AI 分析、同步回飞书）。

### 什么是空白链接？

空白链接是指飞书表格中满足以下条件的记录：
- ✅ 有原始链接
- ❌ 缺少标题、作者或一句话总结（至少缺一项）
- ❌ 处理状态不是"已完成"或"处理中"

**典型场景**：
- 在手机上快速添加视频链接到飞书表格
- 在其他平台（如飞书移动端）添加待处理的链接
- 批量导入链接后需要自动处理

### 方法 3：后台定时运行（推荐生产环境）

#### Windows 系统

**方式 A：直接运行**

双击 `run_scheduler.bat` 或在命令行执行：

```bash
python scheduler_app.py
```

程序会持续运行，默认每 5 分钟检查一次。按 `Ctrl+C` 停止。

**方式 B：使用 Windows 任务计划程序**

1. 打开"任务计划程序"
2. 创建基本任务
3. 设置触发器（如每 5 分钟）
4. 操作：启动程序
   - 程序：`python.exe` 的完整路径
   - 参数：`scheduler_app.py`
   - 起始于：项目目录路径
5. 保存并启用

#### Linux/Mac 系统

**方式 A：直接运行**

```bash
python scheduler_app.py
```

**方式 B：使用 systemd（推荐）**

1. 编辑服务配置文件：
   ```bash
   sudo nano /etc/systemd/system/feishu-scheduler.service
   ```

2. 复制 `feishu-scheduler.service` 的内容，修改路径

3. 启用并启动服务：
   ```bash
   sudo systemctl enable feishu-scheduler
   sudo systemctl start feishu-scheduler
   ```

4. 查看状态：
   ```bash
   sudo systemctl status feishu-scheduler
   ```

5. 查看日志：
   ```bash
   sudo journalctl -u feishu-scheduler -f
   ```

**方式 C：使用 cron**

编辑 crontab：
```bash
crontab -e
```

添加定时任务（每 5 分钟执行一次）：
```
*/5 * * * * cd /path/to/project && /path/to/python check_blank_links.py >> logs/cron.log 2>&1
```

**方式 D：使用 supervisor**

创建配置文件 `/etc/supervisor/conf.d/feishu-scheduler.conf`：
```ini
[program:feishu-scheduler]
command=/path/to/python /path/to/project/scheduler_app.py
directory=/path/to/project
user=your_username
autostart=true
autorestart=true
stderr_logfile=/path/to/project/logs/scheduler_error.log
stdout_logfile=/path/to/project/logs/scheduler.log
```

重启 supervisor：
```bash
sudo supervisorctl reread
sudo supervisorctl update
sudo supervisorctl start feishu-scheduler
```

## 配置说明

### 必需配置

在 `.env` 文件中配置飞书 API：

```env
# 飞书配置
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=xxx
FEISHU_BITABLE_APP_TOKEN=xxx
FEISHU_BITABLE_TABLE_ID=xxx
```

### 可选配置

**修改检查间隔**

编辑 `scheduler_app.py`，修改 `check_interval` 参数：

```python
# 默认 5 分钟（300 秒）
check_interval = 300

# 改为 10 分钟
check_interval = 600

# 改为 1 小时
check_interval = 3600
```

**修改 AI 提供商**

在 `.env` 中配置：

```env
AI_PROVIDER=openai  # 或 gemini
ENABLE_AI_POLISH=true
```

**修改 Whisper 模型**

```env
WHISPER_MODEL=base  # tiny, base, small, medium, large
```

## 处理流程

```
1. 扫描飞书表格
   ↓
2. 识别空白链接
   ↓
3. 更新状态为"处理中"
   ↓
4. 下载视频
   ↓
5. 提取音频
   ↓
6. 语音识别
   ↓
7. AI 分析润色
   ↓
8. 生成 Markdown
   ↓
9. 更新飞书记录
   ↓
10. 状态更新为"已完成"
```

## 常见问题

### Q1: 定时任务没有检测到空白链接？

**检查清单**：
1. 确认飞书表格中有符合条件的记录
2. 检查"原始链接"字段是否有值
3. 检查"处理状态"是否为空或"待处理"
4. 查看日志文件 `logs/app.log`

### Q2: 处理失败怎么办？

**查看错误信息**：
1. 在飞书表格的"错误信息"字段查看
2. 查看日志文件 `logs/app.log`
3. 常见错误：
   - Cookies 过期：重新配置 Cookies
   - API 额度不足：检查 API 余额
   - 网络问题：检查网络连接

### Q3: 如何停止定时任务？

**方法**：
- 直接运行：按 `Ctrl+C`
- systemd：`sudo systemctl stop feishu-scheduler`
- supervisor：`sudo supervisorctl stop feishu-scheduler`
- Windows 任务计划程序：禁用任务

### Q4: 可以同时运行多个定时任务吗？

**不推荐**。多个实例可能会：
- 重复处理同一条记录
- 造成飞书 API 限流
- 资源竞争

建议只运行一个实例。

### Q5: 定时任务会处理已完成的记录吗？

**不会**。定时任务会跳过：
- 处理状态为"已完成"的记录
- 处理状态为"处理中"的记录
- 已有标题、作者和总结的记录

### Q6: 如何调整处理优先级？

目前按照飞书表格的顺序处理。如需调整：
1. 在飞书表格中调整记录顺序
2. 或修改 `core/scheduler.py` 中的排序逻辑

### Q7: 处理速度慢怎么办？

**优化建议**：
1. 使用更快的 Whisper 模型（tiny 或 base）
2. 关闭 AI 润色（`ENABLE_AI_POLISH=false`）
3. 增加检查间隔，减少频繁检查
4. 使用更快的 AI 提供商

### Q8: 如何监控定时任务运行状态？

**监控方法**：
1. 查看日志文件：`logs/app.log`
2. 查看飞书表格的"处理时间"字段
3. 使用 systemd：`systemctl status feishu-scheduler`
4. 使用 supervisor：`supervisorctl status`

## 最佳实践

### 1. 测试流程

在生产环境部署前：
1. 先手动执行一次：`python check_blank_links.py`
2. 检查处理结果是否正确
3. 确认飞书表格更新正常
4. 再启动定时任务

### 2. 日志管理

定期清理日志文件：
```bash
# Linux/Mac
find logs/ -name "*.log" -mtime +30 -delete

# Windows
forfiles /p logs /s /m *.log /d -30 /c "cmd /c del @path"
```

### 3. 错误处理

- 定时任务会自动跳过失败的记录
- 失败记录会标记错误信息
- 可以手动修复后重新处理

### 4. 性能优化

- 合理设置检查间隔（建议 5-10 分钟）
- 避免高峰期处理大量视频
- 使用合适的 Whisper 模型

### 5. 安全建议

- 不要在公共服务器上运行
- 定期更新 Cookies
- 保护 `.env` 文件安全
- 使用独立的飞书应用

## 故障排查

### 日志位置

- 应用日志：`logs/app.log`
- systemd 日志：`journalctl -u feishu-scheduler`
- supervisor 日志：`/path/to/project/logs/scheduler.log`

### 常见日志关键词

- `发现空白记录`：找到待处理的链接
- `处理成功`：记录处理完成
- `处理失败`：记录处理失败
- `飞书同步成功`：数据已同步到飞书
- `飞书同步失败`：同步失败，检查配置

### 调试模式

修改 `utils/logger.py` 启用详细日志：
```python
logging.basicConfig(
    level=logging.DEBUG,  # 改为 DEBUG
    ...
)
```
