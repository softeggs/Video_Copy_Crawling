# iOS 视频情报提取 App 开发计划

## 一、项目定位与架构

### 1.1 产品定位

- **核心价值**：快速收集视频链接，自动提取结构化笔记
- **目标用户**：内容研究者、行业分析师、知识工作者
- **使用场景**：刷视频时一键收藏 → 自动处理 → 随时查看结构化笔记

### 1.2 技术架构

```
┌─────────────────────────────────────┐
│         iOS App (Swift/SwiftUI)     │
│  ┌────────────┐  ┌────────────┐    │
│  │ 快速提交   │  │ 历史记录   │    │
│  │ Share Ext  │  │ 搜索筛选   │    │
│  └────────────┘  └────────────┘    │
│  ┌────────────────────────────┐    │
│  │   本地数据库 (CoreData)     │    │
│  │   (缓存 + 离线支持)         │    │
│  └────────────────────────────┘    │
└─────────────┬───────────────────────┘
              │ RESTful API
              ▼
┌─────────────────────────────────────┐
│      Python 后端服务 (FastAPI)      │
│  ┌────────────┐  ┌────────────┐    │
│  │ 用户认证   │  │ 视频提交   │    │
│  │ (本地Mock)│  │ 记录查询   │    │
│  └────────────┘  └────────────┘    │
│         │                           │
│         ▼                           │
│  ┌────────────────────────────┐    │
│  │  飞书多维表格 (数据源)      │    │
│  │  - 提交新记录               │    │
│  │  - 查询处理结果             │    │
│  └────────────────────────────┘    │
└─────────────────────────────────────┘
              ▲
              │ 定时任务
              │
┌─────────────────────────────────────┐
│       现有处理流水线 (保持不变)      │
│  下载 → 转录 → AI分析 → 飞书更新   │
└─────────────────────────────────────┘
```

### 1.3 数据流设计

1. **提交流程**：App → 后端API → 飞书表格（添加记录）→ 定时任务处理
2. **查询流程**：App → 后端API → 飞书表格查询 → 返回结果 + 本地缓存
3. **离线浏览**：直接读取本地CoreData缓存

### 2.6 本地缓存模块

#### 2.6.1 CoreData数据模型

```swift
// VideoRecord Entity
- record_id: String (主键)
- title: String
- author: String
- url: String
- summary: String
- core_points: [String]
- corrected_text: String
- golden_sentences: [String]
- tags: [String]
- video_type: String
- status: String
- markdown_content: String
- created_at: Date
- processed_at: Date?
- last_synced_at: Date
- is_cached: Bool
```

#### 2.6.2 缓存策略

- **自动缓存**：已完成的记录自动缓存到本地
- **离线可用**：无网络时可浏览缓存记录
- **缓存更新**：每次打开App时同步最新数据
- **缓存清理**：设置中可清除缓存（保留最近7天）

## 三、后端API设计（新增模块）

### 3.1 API服务架构

- **框架**：FastAPI（Python）
- **部署方式**：与现有Streamlit共存，独立端口（如8001）
- **认证方式**：JWT Token（本地Mock验证）

### 3.2 API列表


| 端点                           | 方法   | 功能             | 优先级 |
| ---------------------------- | ---- | -------------- | --- |
| `/api/auth/login`            | POST | 用户登录（本地Mock）   | P0  |
| `/api/videos/submit`         | POST | 提交视频链接         | P0  |
| `/api/videos/list`           | GET  | 获取记录列表（分页、筛选）  | P0  |
| `/api/videos/{id}`           | GET  | 获取单条记录详情       | P0  |
| `/api/videos/{id}/status`    | GET  | 查询处理进度         | P1  |
| `/api/videos/{id}/reprocess` | POST | 重新处理失败记录       | P1  |
| `/api/videos/search`         | GET  | 全文搜索           | P1  |
| `/api/stats/overview`        | GET  | 统计概览（总数、今日新增等） | P2  |


### 3.3 关键实现

- **数据源**：直接查询飞书表格（使用现有`FeishuSync`类）
- **认证Mock**：读取本地配置文件验证账号
- **记录关联**：每个用户对应唯一的`table_id`

**示例代码结构**：

```
api/
├── main.py          # FastAPI主入口
├── auth.py          # 认证模块（本地Mock）
├── videos.py        # 视频记录接口
├── models.py        # Pydantic数据模型
└── deps.py          # 依赖注入（获取当前用户）
```

## 七、服务器部署预留接口

### 7.1 多用户支持预留

当前设计已预留多用户架构，未来服务器部署时需要：

1. **数据库设计**：

```sql
-- 用户表
CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  username VARCHAR(50) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  email VARCHAR(100),
  feishu_table_id VARCHAR(50),  -- 每个用户对应一个飞书表格
  created_at TIMESTAMP DEFAULT NOW()
);

-- 视频记录表（缓存飞书数据）
CREATE TABLE video_records (
  id SERIAL PRIMARY KEY,
  user_id INT REFERENCES users(id),
  record_id VARCHAR(50) UNIQUE,  -- 飞书记录ID
  url TEXT NOT NULL,
  title VARCHAR(255),
  author VARCHAR(100),
  status VARCHAR(20),
  -- ... 其他字段
  created_at TIMESTAMP DEFAULT NOW()
);
```

1. **接口调整**：

- 认证：改为真实数据库验证
- 数据隔离：根据`user_id`过滤数据
- 权限控制：确保用户只能访问自己的记录

1. **飞书表格映射**：

- 每个用户注册时创建独立表格
- 或使用同一表格，通过`user_id`字段区分

### 7.2 接口兼容性

所有API接口已按真实多用户设计，本地Mock只是替换认证逻辑，未来升级时**iOS端代码无需修改**。

## 八、风险与注意事项

### 8.1 技术风险

- **飞书API限流**：实现请求缓存，避免频繁查询
- **数据同步延迟**：处理流程可能需要2-5分钟，需设计合理的轮询策略
- **网络异常**：完善离线缓存和错误处理

### 8.2 用户体验风险

- **处理失败率**：部分视频无法下载，需清晰反馈失败原因
- **等待时间**：长视频处理慢，需设置合理预期

### 8.3 行业规范参考

- **隐私政策**：明确告知数据存储位置（飞书）
- **版权提示**：提醒用户遵守视频平台规则
- **iOS审核**：分享扩展需说明用途
