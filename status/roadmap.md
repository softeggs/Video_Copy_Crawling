# 路线图

- 阶段目标：建立状态中心，固化真实现状，清理“旧后端仍可维护”的错误认知。
- 当前状态：已完成
- 已完成：
  - 建立 `status/` 文档体系
  - 明确后端重建、网页优先、iOS 暂缓的旧判断已失效
  - 建立长期模块分支：`backend/rebuild`、`core/pipeline-engine`、`frontend/web-app`、`ops/streamlit-console`、`ops/status-docs`、`app/ios-client`

## P1 后端可运行

- 阶段目标：让后端认证、视频、管理、健康检查、数据库调度全部可本地运行。
- 当前状态：已完成
- 已完成：
  - 重建 `backend/` 全部基础模块
  - 重建 `core.db_scheduler`
  - 通过 `test_backend_auth.py`
  - 通过 `test_backend_videos.py`
  - 通过 `test_backend_admin.py`
  - 通过 `test_db_scheduler.py`

## P2 iOS 联调优先

- 阶段目标：在不改动现有 iOS App 前端页面和信息结构的前提下，通过最小配置改动完成与统一后端的通信。
- 当前状态：开发中
- 已完成：
  - 后端统一接口、鉴权与视频主链路已完成本地验证
  - iOS 现有项目已具备登录、提交、历史、详情、个人页基础界面
  - iOS 本地开发地址已统一到 `8002`
- 未完成：
  - 将 iOS 默认联调入口从本机回环改为本地局域网可访问地址
  - 清理旧飞书直连和旧 APIService 的历史依赖
  - 仅通过基址、端口和环境配置完成登录、提交、历史、详情、个人页的联调
  - 统一正式模式与测试模式配置入口

## P3 Web 功能优化与界面美化

- 阶段目标：在主链路跑通后，继续补 Web 可用性、交互体验和视觉表现。
- 当前状态：开发中
- 已完成：
  - 建立 `React + Vite + TypeScript` 骨架
  - 实现登录、注册、提交、历史、详情、个人页基础页面
  - 对接统一后端接口
  - 通过 `npm test`
  - 通过 `npm run build`
  - 支持通过 `web_app/.env.local` 切换前端本地后端端口
  - 已补齐提交页 URL 校验
  - 已完成合法视频链接的手动联调验证，确认可成功提交并写库
  - 已完成历史记录分页和状态筛选
  - 已完成提交页、历史页、详情页、个人页的空态、错误态和加载态优化
- 未完成：
  - 增加更完整的交互测试
  - 继续补前端页面级测试与交互覆盖
  - 继续完善失败场景、异常提示和界面美化细节
