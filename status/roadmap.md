# 路线图

## P0 基线固化

- 阶段目标：建立状态中心，固化真实现状，清理"旧后端仍可维护"的错误认知。
- 当前状态：已完成

## P1 后端可运行

- 阶段目标：让后端认证、视频、管理、健康检查、数据库调度全部可本地运行。
- 当前状态：已完成

## P2 iOS 最小联调

- 阶段目标：在不改动现有 iOS App 前端页面和信息结构的前提下，通过最小配置改动完成与统一后端的通信。
- 当前状态：已完成

## P3 下一轮产品能力补齐

- 阶段目标：统一 iOS、Web 与后端的数据模型、记录能力、处理中状态和快捷指令能力。
- 当前状态：**已完成（后端 ✅ + Web ✅ + iOS ✅）**
- 已确定方向：
  - 后端先补统一数据模型与接口
  - Web 与 iOS 共用同一套后端能力
  - 快捷指令采用"独立密钥"方案
  - 不采用"每次提交账号密码"方案
- 调试记录：数据库迁移通过 `ALTER TABLE` 手工修复 P3 字段缺失，Alembic 版本锚定至 `20260327_p3_new_fields`。

## P4 Web 功能优化与界面美化

- 阶段目标：在统一后端能力落地后，先由 Web 接入删除、收藏、处理中状态与快捷指令密钥能力，再推进界面美化。
- 当前状态：**已完成**
- 范围说明：这里的“Web 已完成”指 `web_main.py` 驱动的 Streamlit Web Console；React `web_app/` 仍处于独立推进阶段，未纳入本阶段完成定义。

## P5 iOS 产品能力跟进

- 阶段目标：在 Web 验证完后端接口后，再由 iOS 接入删除、收藏、处理中状态与快捷指令密钥能力。
- 当前状态：**已完成**
- 完成内容：
  - `VideoRecord` 新增 P3 字段（`isFavorited`、`processingStage` 等）
  - `APIService` 新增 7 个 P3 接口
  - `VideoRepositoryProtocol` + 双实现完成
  - `HistoryView` 左滑删除（乐观更新）、星标收藏（乐观更新）、处理中状态展示
  - `ProfileView` 新增"快捷指令密钥"入口
  - `ShortcutKeysView` 完整密钥管理（列表/生成/复制/吊销）

## P6 测试与安全验证

- 阶段目标：补页面级测试、失败场景测试、快捷指令密钥安全验证和越权验证。
- 当前状态：进行中

## P7 服务器迁移准备

- 阶段目标：在 Web 与 iOS 能力完成后，再迁移到服务器。
- 当前状态：进行中
- 现状说明：当前已补本地 Docker 化演练资产、数据库调度常驻入口，完成基础容器联调，并已通过 DeepSeek + 飞书完成真实抖音链接闭环验证；尚未进入真实云主机部署阶段。
