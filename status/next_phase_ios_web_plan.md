# 下一阶段 iOS / Web / Backend 路线表

## 方案结论

| 项目 | 结论 |
|---|---|
| 统一方向 | iOS、Web 共用同一套后端能力与数据模型 |
| 实现顺序 | 后端先补能力，Web 先接入，iOS 后接入，最后做界面美化 |
| 快捷指令方案 | 采用“独立快捷指令密钥”方案 |
| 不采用方案 | 不采用“每次提交账号密码”方案 |
| 安全要求 | 服务端只保存密钥哈希，不保存明文密钥 |

## 统一能力范围

| 能力 | iOS | Web | Backend |
|---|---|---|---|
| 已处理记录显示 | 是 | 是 | 提供统一列表接口 |
| 删除记录 | 是 | 是 | 提供删除接口并同步数据库 |
| 收藏记录 | 是 | 是 | 提供收藏状态字段与接口 |
| 处理中状态详情 | 是 | 是 | 提供阶段、详情、预计时间 |
| 快捷指令提交 | 是 | 是 | 提供独立密钥提交接口 |
| 快捷指令密钥管理 | 是 | 是 | 提供密钥表与管理接口 |

## 后端与数据库路线

### `videos` 表新增字段

| 字段 | 类型 | 用途 |
|---|---|---|
| `is_favorited` | bool | 收藏状态 |
| `processing_stage` | str | 当前阶段：`queued/downloading/transcribing/ai_polishing/syncing/completed/failed` |
| `processing_detail` | str | 人类可读状态说明 |
| `estimated_seconds_remaining` | int \| null | 剩余预计时间 |
| `last_stage_update_at` | datetime \| null | 最近一次阶段更新时间 |

### 新增快捷指令密钥表

建议表名：`shortcut_keys`

| 字段 | 类型 | 用途 |
|---|---|---|
| `id` | int | 主键 |
| `user_id` | int | 关联用户 |
| `key_hash` | str | 密钥哈希值 |
| `key_prefix` | str | 前缀摘要，用于展示 |
| `name` | str | 密钥名称，如“iPhone 快捷指令” |
| `is_active` | bool | 是否有效 |
| `created_at` | datetime | 创建时间 |
| `last_used_at` | datetime \| null | 最近使用时间 |
| `revoked_at` | datetime \| null | 吊销时间 |

## 后端接口路线

### 记录能力接口

| 接口 | 说明 |
|---|---|
| `GET /videos` | 补充返回收藏、处理中状态、预计时间字段 |
| `GET /videos/{id}` | 同样补充收藏、处理中状态、预计时间字段 |
| `DELETE /videos/{id}` | 删除指定记录，并同步数据库删除 |
| `POST /videos/{id}/favorite` | 收藏/取消收藏，建议显式传 `is_favorited` |
| `GET /videos/{id}/status` | 轻量状态接口，供 5 秒轮询 |

### 快捷指令接口

| 接口 | 说明 |
|---|---|
| `POST /shortcut-keys` | 登录态下生成新密钥 |
| `GET /shortcut-keys` | 查看当前用户已有密钥摘要 |
| `DELETE /shortcut-keys/{id}` | 吊销密钥 |
| `POST /shortcut-submit` | 快捷指令专用 URL 提交接口 |

### 快捷指令接口约束

| 规则 | 说明 |
|---|---|
| 只提交 `shortcut_key + url` | 不再提交账号密码 |
| 只允许 URL 提交 | 不授予其他权限 |
| 服务端校验密钥 | 找到对应用户后写入该用户记录 |

## iOS 端路线

### 第一阶段：记录操作

| 功能 | 说明 |
|---|---|
| 已处理记录展示 | 在记录页显示已处理数据 |
| 左滑删除 | 左滑删除某条记录 |
| 删除同步 | 调用 `DELETE /videos/{id}` |
| 星号收藏 | 单条记录支持收藏/取消收藏 |

### 第二阶段：处理中状态

| 功能 | 说明 |
|---|---|
| 状态区域 | 对“处理中”记录新增详细状态区域 |
| 轮询频率 | 5 秒一次 |
| 展示内容 | 当前阶段、当前说明、预计剩余时间 |
| 时间策略 | 预计时间常驻显示，不忽隐忽现 |

### 第三阶段：快捷指令密钥

| 功能 | 说明 |
|---|---|
| 生成密钥 | App 内点击生成快捷指令密钥 |
| 首次展示 | 明文只展示一次 |
| 风险提醒 | 提示密钥是唯一提交凭证，请勿泄露 |
| 密钥管理 | 支持吊销旧密钥、重新生成新密钥 |

### 第四阶段：快捷指令初始化

| 功能 | 说明 |
|---|---|
| 初始化指引 | 指导用户将密钥填入快捷指令 |
| 后续提交 | 每次仅提交 `shortcut_key + url` |

## Web 端路线

### 第一阶段：记录操作

| 功能 | 说明 |
|---|---|

|---|---|

| 详情完整状态 | 详情页展示完整处理中状态区块 |
| 轮询频率 | 5 秒一次 |

### 第三阶段：快捷指令密钥管理

| 功能 | 说明 |
|---|---|
| 生成密钥 | 用户生成快捷指令密钥 |
| 查看摘要 | 展示密钥前缀与使用情况 |
| 吊销密钥 | 废弃旧密钥 |
| 使用说明 | 提供快捷指令初始化说明 |

### 第四阶段：界面美化

| 页面 | 重点 |
|---|---|
| 历史记录页 | 删除、收藏、处理中状态卡片 |
| 详情页 | 收藏、处理中详情、结果展示 |
| 个人页 | 快捷指令密钥管理区 |
| 状态卡片 | 统一视觉与层级 |

## 实现顺序建议

| 顺序 | 模块 | 目标 |
|---|---|---|
| 1 | 后端 | 数据库字段、接口、快捷指令密钥能力先落地 |
| 2 | Web | 先接删除/收藏/处理中状态 |
| 3 | iOS | 再接删除/收藏/处理中状态 |
| 4 | Backend + iOS/Web | 快捷指令密钥能力上线 |
| 5 | iOS/Web | 接入快捷指令密钥入口与管理 |
| 6 | Web | 做界面美化和交互细节 |
