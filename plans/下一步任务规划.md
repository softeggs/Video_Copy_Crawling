## P0 — 立即处理（阻塞性问题）

### P0-1：修复 Kimi API 余额不足问题

**问题描述：**  
Kimi API 账户余额为 -¥0.01，导致定时任务中"原始转录"记录的 AI 润色全部失败（429 错误），系统持续重试但无法成功。飞书表格中目前有 2 条记录卡在"原始转录"状态。

**解决方案（二选一）：**

方案 A — 充值 Kimi 账户（最快）
- 登录 Moonshot AI 控制台充值

方案 B — 切换 AI 提供商（推荐，不依赖充值）
- 在 `.env` 中将 `AI_PROVIDER` 改为 `openai` 或 `gemini`
- 或在 Web 界面侧边栏临时切换提供商，手动重新处理这 2 条记录

**涉及文件：** `.env`、`core/ai_processor.py`

## P2 — 本月完成

### P2-1：后端 — 定时任务稳定性增强

**背景：**  
当前定时任务在 AI 提供商全部失败时会持续重试，日志文件已达 490 万行（极大），存在磁盘占用风险。

**改进点：**
1. **日志轮转**：配置 `logging.handlers.RotatingFileHandler`，限制单文件 50MB，保留最近 5 个文件
2. **失败熔断**：同一条记录连续失败超过 10 次后，自动打上"处理失败"标签，停止重试
3. **余额预检**：定时任务启动前先检查 API 余额，余额不足时跳过 AI 处理并发送告警

**涉及文件：**
- `utils/logger.py`（日志轮转）
- `core/scheduler.py`（熔断 + 余额预检）

**估时：** 3-4 小时

---

### P2-2：后端 — 小红书下载稳定性优化

**背景：**  
小红书频繁更新反爬策略，`cookies_xiaohongshu.txt` 需要定期手动更新，且 yt-dlp 对小红书的支持时常失效。

**改进点：**
1. 下载失败时在飞书记录中写入具体错误原因（当前只写"下载失败"）
2. 添加 Cookie 有效性检测：下载前先验证 Cookie 是否过期，过期时在 Web 界面显示警告
3. 考虑备用下载方案（如调用小红书第三方解析 API）

**涉及文件：**
- `core/downloader.py`
- `utils/platform_detector.py`
- `app.py`（Cookie 过期警告 UI）

**估时：** 4-6 小时

### P2-4：Web 界面 — 批量处理进度优化

**背景：**  
批量处理时，Streamlit 界面的进度反馈较粗糙，用户无法看到每条视频的实时处理状态。

**改进点：**
1. 批量处理时显示每条视频的处理阶段（下载中 / 转录中 / AI 润色中 / 已完成）
2. 失败的条目显示具体错误原因，支持单独重试
3. 批量完成后显示汇总报告（成功 N 条 / 失败 N 条）

**涉及文件：**
- `app.py`
- `core/pipeline.py`

**估时：** 4-5 小时

## P3 — 未来规划

### P3-1：Python FastAPI 后端服务

**目标：**  
当 iOS 用户增多时，将飞书直连模式切换为真实后端 API，iOS App 的 Repository 协议层已为此做好了抽象。

**主要工作：**
- 实现 `POST /auth/login`、`GET /videos`、`POST /videos/submit` 等接口
- 用户系统（每个用户绑定独立飞书表格）
- JWT Token 认证
- 部署到云服务器（推荐：Railway / Render / 阿里云轻量）

**iOS 端改动：**
- 新建 `Services/ServerVideoRepository.swift`（实现 `VideoRepositoryProtocol`）
- `AppConfig.swift` 中将 `useFeishuDirect` 改为 `false`
- View 层**零改动**

**估时：** 2-3 天

---

### P3-2：多用户支持

**目标：**  
当前系统为单用户设计（一个飞书表格），未来支持多用户各自管理自己的视频库。

**主要工作：**
- 用户注册 / 登录系统
- 每个用户创建独立飞书表格（或使用同一表格的不同视图过滤）
- iOS App 登录后展示当前用户的专属数据

---

### P3-3：AI 摘要质量优化

**目标：**  
当前 AI 润色 Prompt 对所有视频类型使用同一套模板，效果参差不齐。

**改进方向：**
- 按视频类型（教程 / 工具推荐 / 评测 / 知识分享）使用不同的 Prompt 模板
- 支持用户在 Web 界面自定义 Prompt
- 增加 AI 输出质量评分机制（基于关键字段完整度）

---

### P3-4：iOS App 推送通知

**目标：**  
用户提交视频 URL 后，处理完成时通过推送通知告知用户。

**技术方案：**
- 需要 Python 后端支持（P3-1 完成后）
- 使用 APNs（Apple Push Notification service）
- iOS App 注册推送权限，后端处理完成后推送
